## Why

源码插件当前已经能够向宿主管理工作台提供页面、插槽、菜单、权限和运行时资源，但缺少面向业务消费端浏览器入口的前端资产托管能力。插件作者如果需要提供独立的消费端页面，只能依赖宿主前端构建或自建静态服务，无法复用插件生命周期、启用状态、版本治理和集群缓存失效机制。

本分支需要补齐的是消费端前端托管能力。消费端前端入口管理`frontend/consumer/`资产、稳定挂载路径、`index.html`缓存治理和治理快照投影，让源码插件消费端页面能够复用宿主已有的插件生命周期、启用状态、版本治理和集群缓存失效机制。

## What Changes

- 为源码插件新增`frontend/consumer/`目录约定，宿主可从本地源码目录或嵌入式文件系统发现消费端前端资产。
- 在`plugin.yaml`中新增`consumer.frontend`声明，用于描述消费端前端挂载路径、入口文件、启用开关和`SPA`回退策略。
- 宿主新增消费端前端资产托管路径和稳定挂载路径解析，支持按插件 ID、版本和相对路径读取资产，也支持通过声明的挂载路径访问入口与静态资源。
- 消费端前端入口的`HTML`响应注入`base`，使干净子路由和相对资源加载能够在稳定挂载路径下工作。
- 消费端前端资产响应使用`ETag`与`Cache-Control`治理，稳定挂载路径下强制重新验证，避免同版本刷新后浏览器长期使用旧入口。
- 源码插件安装、卸载、启用、禁用、升级和集群运行时修订号刷新时，宿主会失效消费端前端挂载索引。
- 新增消费端前端治理快照，投影插件 ID、版本、启用状态、租户治理声明和消费端前端挂载信息。

## Capabilities

### Modified Capabilities

- `plugin-ui-integration`：补充源码插件消费端前端资产托管、稳定挂载路径和缓存治理要求。
- `plugin-manifest-lifecycle`：补充`consumer.frontend`清单声明、资源快照统计和生命周期缓存失效要求。

## Impact

- 影响宿主插件服务、源码插件清单扫描、前端资产 HTTP 托管和插件运行时缓存刷新。
- 不新增数据库 schema、不新增管理 API DTO、不新增前端运行时文案、不新增`manifest/i18n`资源、不新增`apidoc i18n JSON`。
- 数据权限影响：本次只托管插件声明的前端静态资产和治理投影，不新增数据读取、写入、导出或聚合接口。
- 缓存一致性影响：新增进程内消费端前端挂载索引；单机模式通过生命周期本地失效，集群模式复用插件运行时共享修订号刷新时的跨实例失效观察。
- 开发工具与脚本影响：不新增或修改默认开发工具、构建脚本、测试脚本或跨平台入口。
