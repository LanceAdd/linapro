## MODIFIED Requirements

### Requirement:全局字典组件和 Store 必须缓存字典数据

系统 SHALL 提供可复用的`DictTag`组件和 Pinia字典 Store，使字典选项加载一次后跨页面复用，并在字典数据变更后同步已挂载的字典标签。

#### Scenario:DictTag渲染字典值
- **当** `DictTag` 接收字典选项和值时
- **则** 显示匹配的标签和配置的标签样式
- **且** 不匹配的值显示回退文本

#### Scenario:字典 Store 请求去重
- **当** 多个组件并发请求同一字典类型时
- **则** 仅发出一个 API 请求
- **且** 所有调用方共享同一结果

#### Scenario:字典缓存刷新
- **当** 用户刷新字典缓存时
- **则** 缓存的字典数据被清除，后续请求从 API 重新加载

#### Scenario:字典数据变更后已挂载标签同步刷新
- **当** 用户修改某个字典类型下字典数据的标签文字、标签样式或 CSS 类
- **则** 字典 Store 按该字典类型从 API 重新加载选项
- **且** 已经挂载并使用该字典类型的列表标签同步显示最新字典内容
- **且** 刷新过程不得依赖重新打开页面或重新挂载列表组件
