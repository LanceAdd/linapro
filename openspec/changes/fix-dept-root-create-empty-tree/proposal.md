## 背景

组织管理插件的部门创建接口已经支持 `parentId=0` 创建顶级部门，但部门管理页面的新增抽屉将上级部门设置为必选，并且候选项完全来自现有部门树。当 `plugin_linapro_org_core_dept` 中没有任何根部门或部门树为空时，前端没有可选择的上级部门，也不会提交合法的 `parentId=0`，导致无法创建第一条部门记录。

本变更聚焦修复 `linapro-org-core` 源码插件的部门管理前端适配层：在部门选择器中提供稳定的“顶级部门”选项，并为新增部门默认使用 `parentId=0`。不修改 `lina-core` 宿主核心契约、部门后端 API、数据库结构或组织能力边界。
