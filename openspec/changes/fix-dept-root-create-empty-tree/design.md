## 设计说明

本反馈修复属于 `linapro-org-core` 源码插件的前端适配层问题，不改变宿主核心、后端 API、数据库结构或组织能力契约。

部门创建后端已经将 `parentId=0` 定义为顶级部门语义。修复方案是在部门新增抽屉的上级部门 `TreeSelect` 中合成本地“顶级部门”节点，并在工具栏新增场景默认提交 `parentId=0`。真实部门树仍来自现有 `dept/tree` 和 `dept/exclude/{id}` 接口，前端只做展示候选归一化。

为了避免 `dept/exclude/{id}` 的扁平部门 DTO 与 `dept/tree` 的树节点 DTO 在同一控件中显示不一致，抽屉本地将候选数据统一投影为 `id`、`label`、`children` 结构。该归一化只作用于当前页面控件，不扩展为公共抽象。

### 影响判断

- 宿主边界：不修改 `apps/lina-core`。
- API 契约：不新增或修改 HTTP API、DTO、权限标签或响应结构。
- 数据权限：不新增数据访问路径；创建动作继续走现有后端租户过滤和权限校验。
- 缓存一致性：无缓存、快照或失效逻辑影响。
- 数据库：无 SQL、DAO、索引或初始化数据影响。
- i18n：新增插件运行时文案“顶级部门”，维护插件自身 `zh-CN` 与 `en-US` 资源，并提供前端 fallback 以覆盖运行时语言包尚未刷新时的显示。
- 测试：新增插件自有 E2E `TC004`，用接口拦截构造空部门树并断言创建请求提交 `parentId=0`。
